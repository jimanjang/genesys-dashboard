# 📊 [v0.1] 광고_사업팀 운영 대시보드 프로젝트 정리

## 📌 1. 프로젝트 개요
* **목표:** 기존 Generic 콜센터 대시보드를 광고_사업팀(Ads Business Team) 전용 운영 대시보드로 맞춤형 개편
* **주요 요건:** 
  * 당근마켓 광고_사업팀 운영 목적에 부합하는 레이아웃 및 디자인 (당근 서비스 워크 UI/UX 적용)
  * Genesys Cloud API 실시간 연동 (광고 호전환, 광고 노출/성과, 신규 광고 신청 큐 데이터)
  * 지정된 7명의 전담 상담원 그룹 한정 모니터링
  * 실시간 KPI 지표 및 상담원 상태(대화 가능, 유휴, 상호작용 등)의 정확한 동기화

## 🛠️ 2. 기술 스택
* **Frontend:** React, Vite, CSS Grid/Flexbox
* **Backend:** Node.js, Express, Axios
* **Real-time 통신:** WebSocket (Socket.IO)
* **연동 API:** Genesys Cloud Public API (`/api/v2/analytics/conversations/aggregates/query`, `/api/v2/users`)

## 🚀 3. v0.1 주요 구현 내용 명세

### 3.1. 사용자 인터페이스(UI) 개편 
* 기존의 단조로운 화면에서 벗어나 **당근마켓 스타일의 모던한 대시보드 UI**로 전면 개편.
* **상단 헤더:** `당근 서비스 워크 광고_사업팀 운영 대시보드` 타이틀 및 로고 삽입.
* **메트릭 섹션 (CSS Grid 적용):**
  * Top KPI Grid (상단): 실시간 응답률(%), 대기호, 오늘 포기호 위젯 3열 배치.
  * Dual KPI Grid (좌측 하단): 오늘 총 응답/인입호 건수 위젯 배치.
* **큐(Queue) 통계 카드:** `상담중` 및 `상담가능` 상태를 대상 3개 큐별로 분리하여 리스트 형태로 보여주는 전용 컴포넌트(`QueueStatusCard`) 신규 개발.
* **상담원 현황 표 (Agent Table):** 우측 세로형으로 배치하여 실시간 요원 리스트와 요약 상태 배지(초록색/파란색/빨간색) 렌더링.

### 3.2. 실시간 운영 데이터 연동 (Genesys Live Migration)
* **목업(Mock) 데이터 제거:** `genesysApi.js` 내의 임의의 데모 데이터를 걷어내고, .env에 세팅된 Live `Client ID / Secret` 및 `Queue IDs`를 통한 프로덕션 데이터 폴링 구조 확립.
* **큐(Queue) 대상 특정:** 
  * `당근 비즈니스센터_광고 노출/성과`
  * `당근 비즈니스센터_신규 광고 신청`
  * `광고 호전환`
* **KPI 집계 필터링 로직 구현:**
  * 화면 상단의 거시적 지표(응답률, 평균 대기, 포기호 등) 계산 시 **"광고 호전환" 큐 데이터를 제거**하고 순수 광고센터 인입 건만 집계 (요구사항 완벽 반영).

### 3.3. 상담원(Agent) 디테일 모니터링 고도화
* **전담 그룹 필터링:** 큐에 할당된 전체 인원이 아닌, 지정된 담당자 7명 (Binny Byeon, Charlotte Oh, Flora Kim, Kelly cha, Mason Park, Ray Seong, Winnie Go) 만 필터링하여 노출하는 화이트리스트 로직 구현.
* **정렬 로직 일치화:** 실제 운영 대시보드와 동일하게 1차 상태순(대화 가능 우선), 2차 이름 알파벳순(`B -> C -> F ...`) 정렬 로직 확립.
* **정확한 상태(Presence & Routing) 매핑:**
  * Genesys API의 복잡한 원시 상태값을 분석하여 UI 요건에 맞게 세분화.
  * `Available` ➔ `대화 가능` (초록색 배지)
  * `INTERACTING` ➔ `인터랙션 중` / `상담 중` (빨간색 배지)
  * `IDLE` (On Queue) ➔ `유휴` (파란색 배지)
* **실시간 타이머(Duration) 동기화:** 
  * 상담원 상태 유지 시간 표기(`기간` 컬럼)를 위해 API의 `modifiedDate` 대신 라우팅이 시작된 `routingStatus.startTime` 값을 파싱.
  * 실제 운영 화면의 흐르는 시간(예: `2시간 14분 37초`)과 1초 단위로 똑같이 동기화되도록 수정.
